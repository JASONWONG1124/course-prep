Three-way Link · CH 09

Claude Code × Obsidian × 飞书

明确三种工具的分工，把本地知识、Agent 执行和团队交付接起来。

知识小节

x

学习目标

这一节用于理解「Claude Code × Obsidian × 飞书」里的关键概念、使用场景和判断边界。

知识讲解

明确分工

Obsidian 管知识：承载本地 Markdown 文件，帮助人整理、阅读和看见资料关系

Claude Code 干任务：Agent 进入项目、理解目标、调用资料、和大模型对话，Agent 干活

Obsidian 管知识 → Claude Code 干任务 → 飞书接协作和交付

飞书接协作和交付：承接团队共创、对外发布、数据记录、学员或客户交付

Obsidian、claude code、文件夹、项目、知识，都是本地的，没法给被人看，或者团队没法协作，靓靓是自己一个项目，我是自己一个项目，李哥是自己一个项目，那涉及团队协作的怎么办 由于我们用的是不同的项目，而且团队内需要协作，这时候飞书就承担了一个连接的角色

飞书在这个体系里扮演什么角色

这两个工具有一个共同的特征：它们都是本地的

Obsidian、claude code、文件夹、项目、知识，都是本地的，没法给被人看，团队没法协作 靓靓是自己一个项目，我是自己一个项目，李哥是自己一个项目，那涉及团队协作的怎么办 由于我们用的是不同的项目，每个人在自己的本地世界里工作，但团队需要连接，业务需要流转，客户需要拿到东西

飞书承接的四件事

第一：团队共创

团队成员各自在本地用 Claude Code 生产内容，但生产出来之后，需要一个公共的地方汇总、讨论、迭代。

飞书文档支持多人实时协作，你写一段，亮亮补一段，李哥改一段，所有人的工作在同一个地方汇聚。本地产出，云端协作。

第二：对外发布

你用 Claude Code 写好了一份方案、一套课程大纲、一篇内容策划——这些东西要发给客户、发给合作方、发给学员，不能让他们来看你的本地文件夹。

飞书文档可以生成分享链接，对方不需要任何账号就能打开看。本地生产，飞书发布。

第三：数据记录

团队的运营数据、项目进度、客户信息——这些需要多人共同维护、随时更新的数据，放在飞书多维表格里。

每个人本地的项目是自己的，但团队层面的数据在飞书里是共享的、实时同步的。

第四：学员和客户交付

课程资料、学习文档、项目报告——这些交付给学员或客户的东西，通过飞书来承接。

你在本地用 Claude Code 生产，生产完之后发布到飞书，学员在飞书里接收、学习、互动。交付这件事，在飞书里完成。这时候飞书就承担了一个连接的角色

知识小节

飞书 CLI

学习目标

这一节用于理解「Claude Code × Obsidian × 飞书」里的关键概念、使用场景和判断边界。

知识讲解

链接飞书，我们需要通过飞书 CLI 以前，软件和软件之间，几乎是不可能进行联动交互的

但现在 Agent 时代，这个就可以实现了

Claude code 在终端里，飞书也可以被安装到终端里 既然软件和软件之间，不能互通 那就在终端里，手拉手互通

我们只需要安装一个飞书的 CLI，在电脑的终端里 就可以让 claude code 连通飞书了

接着可以演示两个我之前做过的东西 我用 Agent 做好了一个 Claude Code 宝典，虽然是 AI 写的，但里面的表述风格，还有整个宝典逻辑，怎么进行，能通俗易懂讲透，都是我的思路 我在 Agent 里写完了，那咋给群里面的可以观察

飞书 我就让 Claude Code，帮我写入飞书，帮我直接排好版 对此我还特地做了一个排这种富文本的排版的 skill
