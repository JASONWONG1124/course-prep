更好上手的 Agent 我们整个课程的核心工具 主要模型：ChatGPT Codex Claude Code 就是为人而服务，让人干得舒服 就是为AI而服务，让AI干得舒服

毁灭吧我累了

腾讯 字节跳动

前面我们讲的是国外的「御三家」：Claude Code、Codex、Antigravity。

它们的核心特点，是背后都有非常强的模型能力支撑。所以它们不只是界面做得好，而是底层模型真的能处理复杂任务 国内这边也有一些 Agent 产品，比如腾讯的 WorkBuddy，还有字节系的 Trae

但这一页我不会深入讲，因为我对国内 Agent 的判断比较简单：能力不足

Agent 这个东西，表面上看是产品界面，但真正决定上限的是底层模型，还有整个 Agent 的底层框架

一个 Agent 想要真的好用，不只是会打开文件、点击按钮、生成内容，它还要能理解复杂需求，拆解任务，发现问题，执行多步流程，出错以后能自我修正。 这些能力背后，本质上还是模型能力

知识讲解

从模型命名，来感受模型差异

我们先不看 Claude Code 和 Codex 的界面，也不看它们谁按钮更多、谁功能更多。我们先看它们背后的模型

一家公司给自己的产品起什么名字，暴露的是他们内心深处认为这个产品应该是什么东西，应该服务什么样的人，应该在人类生活里扮演什么角色 命名是一家公司最诚实的自我表达 平时真的巨多人来问我，我 xxxxx 要用什么模型，xxxxxx 要用哪个模型 我真的头都大了 …… 说白了就是不知道每个模型的产品定位，能力，风格，才会出现这些问题

先看 Claude 系列 Anthropic 给模型起的名字，全部来自文学和诗歌的世界。 Haiku——俳句。日本的一种极简诗歌形式，通常只有十七个音节，三行。它的美学核心是：用最少的字，捕捉一个稍纵即逝的瞬间，然后让这个瞬间永远定格。一朵花落下来，一片云飘过去，就这样，没了，但你记住了。小，但有重量。快，但有余韵。 Sonnet——十四行诗。西方诗歌里最严谨的形式之一，莎士比亚用它写爱情，弥尔顿用它写信仰。它有固定的结构，有严格的韵律，但在这个严谨的结构里，诗人要把最复杂的情感和思想装进去。形式是约束，但约束里有深度。 Opus——拉丁语，意思是「作品」。但在音乐世界里，Opus 专门用来指一个作曲家最重要、分量最重的那一部作品。贝多芬的 Opus 131，舒伯特的 Opus posthumous——这些不是随便写的东西，是一个创作者穷尽全力的产物。 Fable——寓言。伊索寓言，格林童话，每一个寓言故事背后都藏着一个道理。它用简单的故事包裹深刻的智慧，让所有人都能读懂，但读完之后你会想很久。它是智慧的平民化表达。 Mythos——神话。不是一个故事，是一个文明的世界观底座。希腊神话、北欧神话、中国神话——神话是人类试图解释世界、解释自己从哪来、解释善恶是什么的最宏大叙事。它是文化的根，是集体无意识的载体。

你把这五个名字放在一起看，会感受到人类最古老、最有深度的文化传统 Anthropic 的命名审美，表达的是他们做的不是一个工具，我们做的是一个有文化人格的存在。它懂诗，它懂故事，它懂人类几千年积累下来的智慧和情感。

你跟它说一件事，它不会立刻给你一个答案。它会先感受你在说什么，感受你问题背后的那个真实困惑是什么，然后才开口。它有时候会说「我理解你的意思，但我觉得这个问题可以从另一个角度来看」——它敢有立场，敢不同意你，敢把它真实的判断说出来。

从 模型命名 来感受模型差异、能力与定位

学习目标

这一节用于理解「从模型命名理解能力与定位」里的关键概念、使用场景和判断边界。

知识讲解

就像 iPhone

iPhone 16，就像 Windows

Windows 11。 这种命名方式背后是一种思维方式：这是一个软件产品，它的核心价值是功能和性能，版本号告诉你它比上一版更强

它就比较直男和理工思维，你给它一个任务，它立刻进入执行模式。结构清晰，步骤明确，要素完整，交付物规范。它不会在意这件事背后有什么感受，不会去探究你真正想要的是什么，它只想把你说的这件事做好

它在处理有明确标准答案的任务时表现极好。代码对不对，逻辑通不通，数据准不准——这些有客观判断标准的事情，它做得又快又准 需要感受、需要审美、需要判断「什么样的表达更打动人」的领域，它的局限就出来了。它会给你一个符合标准的答案，但那个答案是没有温度的。

GPT 被训练成极度顺从、极度有用，因为「有用」是它最重要的价值

无非还是那个问题，claude 和 gpt 这两个模型，怎么选 两个不同的模型，代表了两种不同的用途，也代表了两种不同的任务特性

做的是内容，是自媒体，是需要打动人、需要有人味、需要有审美的东西，并且需要一个思考能力 max，具有极强洞察力，把事情干得漂漂亮亮的，那就选 claude

如果你就是拿它干活的，希望 Agent 生态很完整，有控制屏幕，控制鼠标，在浏览器里帮你扒资料，处理 ppt，处理文档，管理资料，做点图，干些不需要高审美的东西，那就选 codex

我之所以会用 claude code 来切入，是基于我们首要的真实场景的 对做自媒体内容的人来说，这意味着什么

你的工作本质上是在做两件事：思考和表达。 思考——你要不断地判断什么内容值得做，什么方向有潜力，你的账号该往哪走，这个选题能不能打动人。 表达——你要把想清楚的东西，用最能触动人的方式说出来。

这两件事，都不是有标准答案的工程问题，都需要感受力、审美力、和真实的人文理解

当然这个世界不是非黑即白的，成年人的世界从来就不是二选一 各有各的特长和特点 像我，两个都是交替着来用的 平时，xxxxxxxx 我就用 claude xxxxxxxxx，我就用 codex

两个 Agent 产品，只是界面和模型性格上有所不同 应用的底层逻辑都是一样的

我们学的都是怎么用好 Agent 的原理，方法，思维和底层逻辑

所以工具的选择，学员后期可以自己去替换或者并行

我不认为我们要从一个最简单，最小白，最傻瓜的软件来入手 Agent 而是要先从一个最适合我们的 Agent 开始

正式打开 Claude Code

网页打开即用，打开一个对话框，开始聊，按对话窗口来分类历史 这是我们一贯用大模型的方式，也是大模型跟我们交互的基础逻辑

我们习惯了这种聊天机器人的产品逻辑

Claude Code 以打开一个文件夹作为项目起始点来开始的 这种方式来源于编程和项目开发，在编程生态里，一个网站，一个 app，是一个文件夹，里面有大量的架构，代码和功能，要开发或者修改一个程序，就是打开这个程序的文件夹来工作的 什么叫项目制？就是你先建一个文件夹，这个文件夹就是它的整个世界
