Agent Workstyle · CH 03

从 Chat 到 Agent 工作方式

把 AI 从聊天窗口推进到文件、终端和真实工作环境里。

知识小节

Agent

学习目标

这一节用于理解「从 Chat 到 Agent 工作方式」里的关键概念、使用场景和判断边界。

知识讲解

一种更前沿的应用方式出现了 把 AI 当成聊天窗口，升级成，把它放进一个项目系统里

可能很抽象，那我们一步步把它全部拆完 说白了就是，帮别人干活的一种人

它在人类世界里早就存在——房产经纪人是你的 Agent，他代你去谈判、去看房、去签合同。演员经纪人是 Agent，代替演员去谈片酬、接资源、处理合约。特工、间谍，也是 Agent，代表某个政府在另一个地方执行任务。 注意——不是帮你想事情，是帮你做事情

把这个概念映射到大模型和 AI，就是帮你把手上的活给干掉的 AI 就是智能体，智能体是 Agent 的中文翻译

它会有很大的区别，以前的是网友，问答机器人 现在的是直接上手完成工作

「举个例子」

谁才是真正的智能体 体之争 智能

封装提示词的智能体 AI Agent 智能体 智能 什么是 智能 - 即智力与能力的总称 从字面上拆解 “智”指感知、认知、记忆和逻辑推理的能力 “能”指在实践中应用知识解决问题、适应环境及创造行动的能力 面对一个没有标准答案的情况，自己判断，自己决定怎么做

知识讲解

它的行为是被提前写死的。你让它写文案，它按照提示词里定义好的逻辑给你输出。它没有在「自主判断」，它在「执行一套固定指令」 提前写好了说明书，你对着它来做就完事了

在它背后写了一套提示词，定义了它的角色、它的规则、它的工作流程。你打开它，问它要选题，它按照那套提示词里定义好的逻辑，给你生成输出。 规则已经定死了它要怎么写了

知识小节

为什么大模型不能看和操作我电脑里的东西，而所谓的 Agent 就可以呢

学习目标

这一节用于理解「从 Chat 到 Agent 工作方式」里的关键概念、使用场景和判断边界。

知识讲解

ChatGPT、Claude、这些大模型，它们不住在你的电脑里

它们住在 OpenAI、Anthropic 的服务器上，那些服务器可能在美国、在某个数据中心机房里，和你的电脑之间隔着整个互联网。

你每次打开网页跟它对话，本质上是你的电脑通过网络，把你打的字发送过去，它在那边处理完，再把回答发回来

它和你的关系，就是网友关系

你网友能通过微信直接看到和操作到你电脑上的东西吗

你有一个网友，学员每天在微信上聊得很好，他很聪明，什么都懂，你遇到问题找他他都能给你出主意。

但有一天你跟他说：「帮我把我桌面上那个文件夹整理一下。」 他能做到吗？ 他在他家，你在你家。他看不到你的桌面，摸不到你的文件夹，哪怕他再聪明，他也没有办法穿越物理距离伸手进你的电脑里操作东西。

Agent 解决这个问题 大模型是远端的能，本地程序是本地的

它有权限读你的文件、写你的文件、打开软件、执行命令、操作你的系统

大家想一下，我们依赖电脑吗？所谓的工作，绝大多数都是在电脑里 电脑里，有我们的 xxx，xxxx，xxxxx

电脑终端 是什么

我们平时用的电脑 是什么样的

你想打开一个文件，双击它。你想删除一个东西，拖进垃圾桶。你想调整音量，拖一个滑块。 这一切——桌面、图标、窗口、按钮——都是给人类设计的视觉包装 哪里不会就点哪里，已经弱智得像学习机了

它存在的唯一目的，是让不懂电脑的人也能用电脑

但在这层包装的下面，电脑真实的样子是什么？

知识小节

它是电脑的超级中枢核心

学习目标

这一节用于理解「从 Chat 到 Agent 工作方式」里的关键概念、使用场景和判断边界。

知识讲解

终端是 Agent 进入电脑底层操作环境的入口。macOS 可以通过「终端」打开，Windows 可以通过 PowerShell 或命令提示符打开。

学员能看到的这个黑框，就是终端 没有图标，没有按钮，没有鼠标可以点。只有一行光标在等你输入文字

你用文字告诉它做什么，它就做什么

看不懂是吗？门槛很高对吧 想象一下，如果你是一个普通人，买一台电脑回家，打开就是这样的，你用得明白吗，那我情愿不用 想要被极大程度范围被传播和普及的唯一准则，把它做得弱智「豆包 和 Claude 的对比」 为什么 claude code 的用户人数全球不超过 1000 万，但是豆包随随便便就几个亿

你平时用鼠标能做的所有事情，在终端里都能做，而且能做得更彻底、更精准、更快。

试一下 echo 「你好，世界」 > 「$HOME/Desktop/ 线下课演示 .txt」 osascript -e 「set volume output volume 80」

在终端里，你可以一条命令，同时处理一万个文件。 桌面是给普通用户用的，终端是给真正掌控这台电脑的人用的

关闭、新建、拖动、下一步、返回上一层，这些按钮，本身并没有功能的 按钮被按下的那一刻，其实是把一个命令发送给了终端，让终端去执行的

而 Claude Code，就是在终端里工作的

它通过终端，读你的文件、读你的选题，读你的 IP 资料，帮你写文档、执行命令、安装工具、操作你电脑上的任何东西。 终端是它的工作台，是它真正能触碰你电脑世界的入口。

你安装 Claude Code，打开的第一个东西就是终端。不是因为要故意搞得很技术，而是因为终端就是电脑真正的操作层，Agent 要干活，必须从这里进来。

知识小节

Claude Code 界面

学习目标

这一节用于理解「从 Chat 到 Agent 工作方式」里的关键概念、使用场景和判断边界。

知识讲解

第一眼看上去，它就是一个黑色的终端窗口，没有任何花哨的东西。这不是它的缺陷，这就是它的设计哲学

它不需要好看，它需要好用，直接 直接终端运行就是没有中间商赚差价

它确实有学习成本，它不像网页那样一眼就知道怎么用。

但这种别扭的感觉是暂时的，用顺了之后你回不去，就像你用了智能手机之后回不去按键机一样 原生界面，适合想要最大能力、愿意接受学习曲线的人。

Claude Code 界面 VS Code 插件 - 更图形化的 Claude Code 体验 信息区 对话区 输入框

这种最原生的命令行的使用方式 VS Code 插件 - 更图形化的 Claude Code 体验 Claude Code 界面 - 更符合现代人的使用习惯 - 能看到文件夹 - 跟以往用的大模型方式类似，更好上手 - 各功能分区，可以拖动窗口，能同时对话和看文件 优点 缺点 - 非原生应用，体验没有终端原生丝滑 - 没有很多原生的功能

文件管理区 窗口卡切换 对话区 文件阅读区

知识小节

除了它就没有其他AI Agent工具了吗

学习目标

这一节用于理解「从 Chat 到 Agent 工作方式」里的关键概念、使用场景和判断边界。

知识讲解

商业竞争会快速跟进，单个工具的优势很难长期独占。

Anthropic OpenAI Google 腾讯 字节跳动 AI AGENT 产品选择 国外 御三家 国内 大厂 Claude ChatGPT Gemini CHINA all 主要模型

Claude Code 是 Anthropic 出的 Agent，也是我们整个课程的核心工具

它背后的模型是 Claude，在处理复杂任务、长文本理解、以及内容创作这几个维度上，目前综合表现是最好的 公认最强的 Agent

第一个 Agent 产品就是 Claude 做的

我们明天学的 skill，也是他们发明的

用它的主要方式就命令行，还有 Vs code 里面的插件，还有一个在官方的 claude desktop 的软件里面用

Claude code 的产品设计哲学就是，能力优先 不是 Anthropic 不会做图形界面，是他们主动选择不做

普通软件界面是给人设计的： 打开页面、点按钮、选菜单、拖拽文件。

Agent 不是这样工作的，Agent 更像一个执行任务的人，它需要连续做很多步： 不停的看文件 找文件 → 找问题 → 改内容 → 运行命令 → 看结果 → 再修改 → 再验证 这些动作在终端里最顺

所以终端对普通人不友好，但对 AI 来说很友好

Claude code 就是为了 AI 而服务，让 AI 在它的舒适区里干得舒服

Codex 的产品性格会不太一样，降低使用门槛

它一诞生就是冲着更广的受众去的，所以首先就给每个人设计了一个图形界面 更像「Agent 时代的豆包逻辑」： 不是服务少数高手，而是让更多普通人能开始使用

这个逻辑就有点像 Claude 和豆包的区别。

Claude 更强，更专业，但使用门槛更高。 豆包的定位不是服务少数高手，而是希望尽可能多的人都能用上 AI。 Codex 在 Agent 产品里，也有一点类似的方向。

广度优先的产品，必须牺牲一部分深度控制感 换来更强的普适性和上手体验

Claude Code 追求深度，Codex 追求广度。 深度产品的问题是门槛高 广度产品的问题是不能把所有能力都摊开给你
