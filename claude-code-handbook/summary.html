把当天成果带回真实业务，在使用中持续更新系统。

知识小节

今天带回去的是一个刚出生的系统

学习目标

这一节用于理解「总结、答疑与课后行动」里的关键概念、使用场景和判断边界。

知识讲解

今天我们做完了最重要的一件事——把地基打好了。

你有了业务地图，有了项目架构，有了初始的 CLAUDE.md，有了第一批资料。这个系统已经存在了，它已经能跑了。

这里需要先明确一个判断：

你今天带回去的，是一个刚出生的系统，不是一个成熟的系统。

它现在的样子，就像你刚开始做账号的第一天——框架在，但内容很少，经验很薄，还有很多地方不顺手。 没有人能在一天之内建出一个超级好用的知识库。我自己的知识库，是我用了一年多，每天在用、每天在更新、每天在修正，才长成现在这个样子的。

它是养出来的，不是搭出来的。 罗马也不是一天能建成的

所以回去之后最重要的事情只有一件：开始用它。 不要等它完美了再用。它永远不会完美，它只会在你用的过程中越来越好。 每一次你让 Agent 帮你干活，它的输出质量会告诉你知识库还缺什么。缺什么就补什么，补进去之后下次就好一点。 每一次你发现它做错了、理解偏了、输出不是你想要的，就把这个问题变成一条规则写进 CLAUDE.md。规则越来越多，它越来越懂你。 每一次你有了新的判断、验证了新的规律、踩了新的坑，就把它写成一个文件放进知识库。文件越来越多，它调用的时候越来越准。

这个系统和你之间的关系，不是你在用一个工具。 是你在养一个越来越了解你的协作者。

你投入得越多，它回报你的就越多。你今天放进去的每一份资料、写进去的每一条规则，都是在让这个系统变得更懂你一点点。 时间越长，这个优势越大。

六个月后，你的系统和你合作伙伴的系统，差距会是肉眼可见的——因为里面装的是你这六个月真实积累的所有判断和经验，这是别人复制不了的东西。 今天是第一天。开始用它。

Claude Code 小技巧 答疑环节

很多人用 Claude Code，还是把它当成一个更强的聊天窗口。 我问，它答；我让它改，它改；我让它查，它查。

但 Claude Code 里有一个很重要的能力，叫 SubAgent，也就是子 Agent。

你可以把它理解成：主 Agent 可以临时派出几个小弟，在后台自己单开窗口，去做不同的事情。 每个子 Agent 有自己的任务、有自己的上下文、有自己的观察范围。

它们做完以后，不是直接把你的主窗口搞乱，而是把结果汇总回来，交给主 Agent 判断。

它解决的第一个问题：节省主上下文
